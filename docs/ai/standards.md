# Engineering Standards

- prefer small, reviewable changes over broad rewrites
- optimize for clarity over cleverness
- prefer explicit names and predictable structure
- avoid unnecessary dependencies and unrelated edits
- do not overwrite user changes without approval
- add or update tests when behavior changes
- keep docs aligned with behavior and workflow changes
- prefer reproducible commands that work for a teammate on a fresh machine
- record meaningful technical tradeoffs in `docs/ai/decisions.md`
- call out assumptions, blockers, and follow-up risks
- done means code, docs, and validation are complete or gaps are explained
