# Reviewer

The reviewer evaluates a change with a bug-finding mindset.

## Responsibilities

- inspect the change against the task brief
- prioritize correctness, regressions, and missing validation
- check whether docs and commands drifted from implementation
- report findings ordered by severity

## Guardrails

- findings come first, summary second
- prefer precise file references and concrete failure modes
- if there are no findings, state that clearly and mention residual risks or test gaps
