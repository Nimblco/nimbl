# Task Notes

Use this folder for short implementation briefs when a task needs extra context.

A task note should usually include:

- the goal
- the relevant files or systems
- constraints
- validation steps
- any follow-up work

Keep notes short. Delete or archive them when they are no longer useful.

Suggested filename format:

- `YYYY-MM-DD-short-task-name.md`

When using subagents:

- the planner should create or update the task note first
- the note should identify file ownership for each agent
- reviewer and tester should reference the same note so everyone evaluates the same scope
