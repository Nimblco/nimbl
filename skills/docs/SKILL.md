# Documentation Rules

- Keep docs concise, scannable, and consistent with the codebase.
- When architecture, workflow, or behavior changes, update the relevant files in `docs/ai/`.
- Prefer documenting decisions and tradeoffs in `docs/ai/decisions.md`.
- Create or update a spec in `docs/specs/` and a plan in `docs/plans/` before implementation when work changes behavior, architecture, workflow, or spans multiple steps.
- Keep spec, plan, and task brief filenames aligned around the same date and topic when practical.
- Do not overwrite user changes without approval.
