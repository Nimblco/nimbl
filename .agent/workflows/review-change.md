# Review Change

Use `docs/ai/subagents/reviewer.md` and the current task brief.

Review the current changes with a bug-finding mindset.

Checklist:

1. check correctness and likely regressions
2. check missing tests or weak validation
3. check docs drift versus code changes
4. check risky assumptions, coupling, and operational impact

Return findings first, then open questions, then a short summary.
