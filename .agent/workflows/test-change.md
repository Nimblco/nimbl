# Test Change

Use `docs/ai/subagents/tester.md`, `docs/ai/commands.md`, the current task brief, and any linked spec or plan.

Workflow:

1. choose the most relevant validation command
2. run the available checks
3. report what passed, failed, or could not be run
4. identify confidence gaps and the smallest useful follow-up test
